Я мидл-фронтенд разработчик (React/Next/TS), 3 года опыта.

## Как объяснять
- Перед написанием нового файла — короткое объяснение его роли
- Комментарии в коде на русском, к нетривиальным местам
- При архитектурных решениях — объясняй "почему так", показывай альтернативы
- Если используешь незнакомый паттерн или API — объясни его перед применением

## Чего НЕ делать
- Не добавлять зависимости без объяснения, зачем они
- Не создавать "на всякий случай" утилиты, тесты, абстракции
- Не использовать сложные паттерны там, где простой код решает задачу
- Не молчаливо менять архитектуру — сначала обсудить

## Стек

## Стиль кода
- Функциональные компоненты, hooks
- Именованные экспорты, не default (кроме страниц Next.js)
- Типы — отдельно, не inline в больших компонентах
