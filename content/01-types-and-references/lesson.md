# Типы данных: значения и ссылки

В JavaScript восемь типов данных. Семь из них — **примитивы**, и один — **объект**. Разница между ними определяет, как переменные ведут себя при копировании, сравнении и передаче в функции. Это фундамент, на котором держится почти всё остальное в языке.

> Ключевая идея: примитивы копируются по значению, объекты — по ссылке. Если ты понимаешь это одно предложение по-настоящему, ты избегаешь целого класса багов.

## Восемь типов

Семь примитивов: `string`, `number`, `boolean`, `null`, `undefined`, `bigint`, `symbol`. И один ссылочный тип — `object` (к нему относятся обычные объекты, массивы, функции, даты — всё, что не примитив).

Примитивы — это простые неделимые значения. Объект — это структура, которая может содержать другие значения и менять своё содержимое.

> Проверить тип можно через `typeof`. Осторожно: `typeof null` возвращает `"object"` — это известный баг языка, который оставили навсегда ради обратной совместимости.

Примитивы (`a`, `s`) хранят значение прямо в себе. А переменная `obj` хранит не сам объект, а **ссылку** — указатель на место в куче, где объект реально лежит. Это маленькое различие порождает всё остальное.

## Копирование: вот где начинаются баги

Когда ты присваиваешь одну переменную другой, поведение зависит от типа.

Для примитивов копируется само значение — две независимые копии:

```js
let a = 10;
let b = a;   // копируется значение 10
b = 20;
console.log(a); // 10 — a не изменилась
```

Для объектов копируется **ссылка**, а не объект. Обе переменные указывают на один и тот же объект в куче:

```js
let obj1 = { x: 1 };
let obj2 = obj1;   // копируется ссылка, не объект
obj2.x = 99;
console.log(obj1.x); // 99 — изменился тот же объект!
```

Это и есть самый частый источник багов у джунов: «я скопировал объект и поменял копию, а оригинал тоже изменился». На самом деле копии не было — была вторая ссылка на тот же объект.

## Сравнение

С тем же принципом связано сравнение. Примитивы сравниваются по значению, объекты — по ссылке:

```js
"hi" === "hi";        // true — значения равны
{ x: 1 } === { x: 1 } // false — это два РАЗНЫХ объекта в куче

let a = { x: 1 };
let b = a;
a === b;              // true — одна и та же ссылка
```

> Два объекта равны через `===` только если это буквально один и тот же объект (одна ссылка). Одинаковое содержимое не делает их равными.

## Как сделать настоящую копию объекта

Если нужна независимая копия, ссылку копировать недостаточно. Варианты:

Поверхностная копия (копирует верхний уровень, вложенные объекты остаются общими ссылками):

```js
let copy = { ...original };           // спред
let copy2 = Object.assign({}, original);
```

Глубокая копия (копирует всю вложенность):

```js
let deep = structuredClone(original);  // современный способ
```

> Поверхностная копия копирует только первый уровень. Если внутри объекта есть вложенный объект — он по-прежнему общий между копией и оригиналом. Это ловушка, про которую спрашивают на собесах.

## Передача в функции

Аргументы в JavaScript всегда передаются по значению — но для объектов «значение» это и есть ссылка. Поэтому функция может изменить содержимое переданного объекта:

```js
function mutate(o) {
  o.x = 100;   // меняет оригинал — у нас та же ссылка
}
let obj = { x: 1 };
mutate(obj);
console.log(obj.x); // 100

function reassign(o) {
  o = { x: 999 }; // НЕ меняет оригинал — переназначили локальную ссылку
}
reassign(obj);
console.log(obj.x); // 100 — осталось как было
```

Разница тонкая, но важная: менять **содержимое** объекта через ссылку — меняет оригинал. Переназначать саму переменную внутри функции — не меняет.

## Что важно вынести

Примитивы живут как значения и копируются целиком — изменения независимы. Объекты живут в куче, а переменные хранят ссылки на них — копирование переменной копирует ссылку, а не объект. Сравнение объектов через `===` проверяет тождество ссылок, не содержимое. Для независимой копии нужен спред или `structuredClone`. В функции содержимое объекта изменяемо через ссылку, но переназначение переменной локально.
